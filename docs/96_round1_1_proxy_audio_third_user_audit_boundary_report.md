# `round1.1 / proxy-audio third user audit boundary` 报告

## 目的
- 记录第三轮用户试听结论
- 判断去音调化后的 `proxy audio` 是否已经足以支持 branch ranking
- 明确当前人工审核链路的有效边界

## 第三轮用户试听结论
用户对第三轮试听包的反馈是:
- 三个版本的音调已经一致
- `D22` 和 `D29` 之间的区别几乎不可区分
  - 只能勉强给出误差级别判断:
    - `D29` 相比 `D22` 略显柔和
- `D33` 明显更不稳定
- 但由于当前代理音频与原音频结构相差仍较大，
  无法直接对照原音频判断:
  - 这种不稳定是不是更真实地反映了原音频
  - 还是只是代理重建器的副产物

## 当前可确认结论
这轮试听已经足以确认:
1. 去音调化修正是有效的。
   - 当前 bundle 不再主要被“谁更高音”主导。
2. 在当前 proxy 路线下，`D22` 与 `D29` 基本不可稳定区分。
   - 即便存在差异，也只到“略柔和”这种误差级判断。
3. `D33` 在当前代理音频里表现出更明显的不稳定性。

## 当前不能确认的结论
这轮试听仍然不能确认:
1. `D33` 的不稳定是否代表它更贴近原音频中的真实结构变化。
2. `D33` 的不稳定是否只是代理重建器对该分支的放大失真。
3. `D22` 与 `D29` 是否在真实 runtime 音频里存在比当前 proxy 更可听的差异。

## 当前边界判断
到这一步，当前 `proxy audio` 路线的能力边界已经比较清楚:
- 能做的:
  - 粗结构审核
  - 发现明显的稳定性异常
  - 阻止把单纯数据 leader 直接升为最终最优解
- 不能稳定做的:
  - `D22 / D29` 这种细粒度 branch ranking
  - 与原音频做可信的一一结构真实性对照
  - 最终音质判断

## 当前结论
1. 当前 `proxy audio` 已经达到“最低限度的人工 gate”作用。
2. 但它不足以承担正式的细粒度分支判优。
3. 现阶段可保留的人工试听结论是:
   - `D22 / D29` 听感上基本打平
   - `D33` 更不稳定
4. 若后续要判断:
   - 哪个更接近真实目标结构
   - 或 `D22` 与 `D29` 的细粒度差异
   则需要继续补更接近 runtime 的 synthesis / vocoder 链路。
