## 二、需要用户做判断的决策点
### 决策点 A：目标文本中的标点如何处理
当前观察：
- 标点频率高，明显参与句内停连和语气。
- 文本来自游戏提取，保留原标点可能更贴近实际演出节奏。

可选方案 1：完整保留原标点
- 优点：
  - 最大限度保留原始停连和语气信息。
  - 更利于后续做语气、事件和韵律相关分析。
- 缺点：
  - 文本规范化和对齐难度更高。
  - 部分标点可能需要特殊词表或额外清洗规则。

可选方案 2：保留主要停顿标点，归一化其余特殊符号
- 优点：
  - 在保留基本节奏信息的同时，降低文本分布复杂度。
  - 更利于构建稳定的训练词表和前处理规则。
- 缺点：
  - 会损失一部分演出风格信息。
  - 需要定义“主要标点”和“特殊符号”的映射规则。

可选方案 3：训练前全部去标点
- 优点：
  - 文本分布最简单。
  - 对基础内容建模实现成本最低。
- 缺点：
  - 丢失停连和语气线索。
  - 后续若要恢复句子节奏，往往要额外补规则或事件建模。

当前建议：
- 更偏向方案 2。
- 理由：
  - 当前数据里的标点信息确实有价值，但全保留会给首版预处理和对齐增加不必要复杂度。
  - 先保留主停顿类标点、归一化特殊标点，更利于首阶段工程推进。

### 决策点 B：源说话人录音的静音裁剪强度
当前观察：
- 以 100 ms 窗看，`-40 dBFS` 以下窗占比约 81.90%，静音很多。
- 但这只是幅度统计，不等于全部都是“可删静音”；其中可能混有呼吸、环境底噪和长停顿。

可选方案 1：激进裁剪
- 参考思路：
  - 使用更高阈值，例如接近 `-35 dBFS`。
  - 重点追求尽快去掉大部分静音。
- 优点：
  - 可快速缩短有效数据时长，减少后续对齐和训练浪费。
  - 更适合尽快构建第一版切分结果。
- 缺点：
  - 容易误删弱起音、轻声和尾音。
  - 更容易把真实语音边界切坏。

可选方案 2：中等裁剪
- 参考思路：
  - 以 `-40 dBFS` 左右作为起点，再加最小时长约束和边界缓冲。
- 优点：
  - 在“去大块静音”和“减少误删”之间相对平衡。
  - 更适合第一版可控试运行。
- 缺点：
  - 仍需复核一批样本。
  - 对办公环境底噪的鲁棒性有限。

可选方案 3：保守裁剪
- 参考思路：
  - 从更低阈值开始，例如接近 `-45 dBFS` 或更低。
  - 只去掉最明显的长静音段。
- 优点：
  - 更不容易伤及真实语音内容。
  - 适合先观察源录音的真实噪声底线。
- 缺点：
  - 会保留更多无效静音。
  - 后续训练和人工检查成本更高。

当前建议：
- 更偏向方案 2。
- 理由：
  - 当前源录音静音比例太高，保守裁剪收益有限。
  - 激进裁剪风险也明显偏大，尤其在办公场景录音里容易误删弱语音。

### 决策点 C：目标数据的采样率和声道如何统一
当前观察：
- 目标数据里有 46 条不是 44100 Hz。
- 其中 5 条是双声道。
- 如果不先统一，后续特征提取和训练输入格式会不稳定。

可选方案 1：全部转换为统一训练副本
- 参考思路：
  - 训练前统一重采样到单一采样率，并统一转单声道。
  - 原始数据保留不动，转换结果单独放目录。
- 优点：
  - 工程最稳。
  - 训练和预处理代码最简单。
- 缺点：
  - 会引入一次额外重采样。
  - 需要额外存储空间。

可选方案 2：保留原始数据，按原采样率分桶处理
- 优点：
  - 尽量少改动原始信号。
  - 更利于后续分析不同来源的分布差异。
- 缺点：
  - 预处理、缓存和训练流程复杂很多。
  - 对当前 MVP 阶段性价比不高。

可选方案 3：统一主流样本，隔离少数异常样本
- 参考思路：
  - 先保留多数样本进入统一训练副本。
  - 少数低采样率或双声道样本单独标记，后续决定是否纳入。
- 优点：
  - 可以最快推进主流程。
  - 异常样本不会立刻污染整体流程。
- 缺点：
  - 短期内会牺牲一小部分数据覆盖率。
  - 后续仍要再处理这批隔离样本。

当前建议：
- 更偏向方案 3 起步，然后在训练副本层面逐步收敛到方案 1。
- 理由：
  - 先把主流样本跑通，比一开始就做全量分桶更务实。
  - 少数异常样本值得保留记录，但不值得在 MVP 早期拖慢全链路。

## 三、当前建议的后续动作
1. 由用户确认以上三个决策点的方向。
2. 确认后实现标准化清洗管线，并把“原始数据”和“训练副本”彻底分离。
3. 对源录音高峰值时间段做二次检查，判断其中哪些属于可接受语音峰值，哪些更像突发噪声。
4. 在标准化 manifest 中加入：
  - 文本规范化字段
  - 采样率/声道标准化字段
  - 静音裁剪前后时长
  - 异常标记
