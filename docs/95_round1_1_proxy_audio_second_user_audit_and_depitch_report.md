# `round1.1 / proxy-audio second user audit and depitch` 报告

## 目的
- 记录第二轮用户试听反馈
- 判断第二版 `proxy audio` 是否已经足够支撑分支比较
- 若仍被频率伪差异污染，则继续去音调化

## 第二轮用户试听结论
用户对第二版试听包的反馈是:
- 这一版听起来像上一版啸叫的低频版
- 停顿和音量结构较为明显，已经能和原音频建立关联
- `D22` 与 `D29` 的区别仍然细微到无法稳定区分
- `D33` 的频率 / 音调明显更高
- 除此之外，没有进一步结论

这说明:
- 第二版已经比第一版更接近“可听结构”
- 但当前 bundle 仍然被可听音调差异污染
- 因此它仍不适合直接拿来做正式的分支胜负判断

## 当前解释
这轮试听里真正有效的信息是:
- 用户已经能听出:
  - 停顿
  - 音量结构
  - 与原音频的粗对应关系

这轮试听里仍不应直接拿来判优的信息是:
- `D33` 的更高频率 / 更高音调
  - 这更像代理重建器副产物
  - 不是当前任务真正想比较的结构优势

## 本轮处理
已继续收紧 `proxy audio` 重建规则:
- 统一可听 carrier frequency
- 不再允许不同分支通过 audible pitch 形成主导差异
- `zero_cross` 仅保留为较弱的 brightness / noise mix 控制

本轮目标很明确:
- 让试听更集中在:
  - 停顿
  - 能量起伏
  - 边界收束
  - 稳定性
- 而不是集中在:
  - 谁更尖
  - 谁更高
  - 谁更像带音调的合成器

## 当前结论
1. 第二版 `proxy audio` 依然不能支持可靠的 branch ranking。
2. 它的有效作用只到“让用户开始听到结构轮廓”为止。
3. 当前已经继续执行去音调化修正，并将重新导出 `D22 / D29 / D33`。
4. 在用户确认前，仍不把任何数据 leader 直接升为当前最优解。
