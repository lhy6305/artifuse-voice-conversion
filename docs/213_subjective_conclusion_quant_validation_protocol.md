# 213. 主观结论量化回查规范

## 背景
- 本项目当前已多次出现:
  - 人耳先得到某种直觉
  - 后续再发现:
    - 片段太短
    - target 本身携带瞬态
    - 指标解释方向错了
    - 或 tradeoff 被误写成单边胜负
- 因此从本轮开始，
  任何主观结论
  都不应只以
  “我听起来像”
  直接落盘，
  而应尽量回到
  可复查、可重复的量化证据上

## 规范目标
1. 降低:
   - 错觉
   - 心理暗示
   - 先入为主
   - 选择性记忆
   对结论的污染
2. 让主观听审
   变成:
   - 提出假设
   - 标记风险
   - 触发量化验证
   的入口，
   而不是最终裁决本身
3. 保证后续接班时，
   能分清:
   - 哪些只是听感猜测
   - 哪些已被量化回查支持

## 适用范围
- 适用于之后的所有主观结论
- 也适用于之前已经写下、
  但还没有量化回查支撑的主观结论
- 包括但不限于:
  - 毛刺更多 / 更少
  - 更稳定 / 更不稳定
  - 更尊重原音量变化
  - 静音更干净
  - 边界更自然
  - 节奏更顺
  - 更像 target

## 主观结论分级

### A. 听感观察
- 定义:
  - 单次或少量听审里出现的直觉
- 允许写法:
  - “怀疑”
  - “疑似”
  - “需要量化回查”
- 禁止写法:
  - “已证明”
  - “确认更好/更差”

### B. 听感趋势
- 定义:
  - 多条样本重复出现，
    但量化仍未完成
- 允许写法:
  - “当前人耳趋势更偏向”
  - “尚待量化确认”
- 禁止写法:
  - “系统性更优/更差”

### C. 量化支持的结论
- 定义:
  - 已存在与主观方向一致的
    可重复量化指标
  - 且至少完成一次
    回查
- 允许写法:
  - “当前量化与人耳一致地表明”
  - “在当前样本范围内更可能成立”

### D. 强结论
- 定义:
  - 主观与量化一致
  - 样本覆盖足够
  - 已排除主要混淆因素
- 允许写法:
  - “当前可作为阶段性主结论”
- 备注:
  - 这一级应少用，
    默认不要轻易写

## 最低量化回查要求

### 1. 任何主观结论都必须先翻译成可测假设
- 例:
  - “`step72` 毛刺更多”
    要翻译成:
    - 哪类窗口
    - 哪种毛刺
    - 用什么指标表示
- 例:
  - “`step60` 静音底音更明显”
    要翻译成:
    - 低活动段里
      活动残留比例是否更高

### 2. 必须明确对应的量化代理指标
- 推荐口径:
  - 毛刺 / 断续:
    - `fragmentation_score`
    - `extra_bursts`
    - `activity_toggle_mean`
    - `sample_delta_peak`
  - 静音底音泄漏:
    - `mean_active_fraction`
  - 尊重原音量变化:
    - candidate activity
      与 target activity
      的相关度 / 偏差
    - 低活动段和高活动段
      的动态范围保真度
- 若现有指标不够，
  先补指标，
  不要直接跳过量化回查

### 3. 必须记录样本范围和不适用样本
- 每次量化回查都要写清:
  - 看了哪些记录
  - 排除了哪些记录
  - 为什么排除
- 尤其要单独标记:
  - target-correlated
    的 breath / sigh / 爆破 / 清辅音窗口

### 4. 必须记录主观与量化是否一致
- 至少分成三类:
  - 一致
  - 部分一致
  - 冲突
- 若冲突，
  不允许继续写强主观结论

## 主观与量化冲突时的默认规则

### 1. 默认降级措辞
- 一旦冲突，
  结论自动从:
  - “确认”
  降级为:
  - “怀疑”
  - “趋势”
  - “待回查”

### 2. 默认优先找混淆因素
- 先检查:
  - 片段是否太短
  - 是否贴近音频边界
  - target 本身是否已有瞬态
  - 是否只看了单一指标
  - 是否把 tradeoff
    错写成单边输赢

### 3. 默认补一轮定向量化
- 不是继续扩大主观讨论，
  而是补:
  - 更纯净样本
  - 更长窗口
  - 更贴近该问题的指标

## 对过去结论的追认规则

### 1. 过去仅由听感支撑的结论
- 默认归类为:
  - 听感观察
  - 或听感趋势
- 不再直接当作
  已确认结论引用

### 2. 若后续量化支持
- 才升级为:
  - 量化支持的结论

### 3. 若后续量化冲突
- 就要回写文档，
  明确说明:
  - 先前结论为何不稳
  - 当前更准确的解释是什么

## 执行要求
1. 之后新增任何听审结论，
   默认同时写:
   - 主观观察
   - 对应量化假设
   - 已有量化证据
   - 尚缺的量化证据
2. 若暂时没有量化证据，
   结论只能停留在:
   - 听感观察
   - 听感趋势
3. 若用户明确指出
   某主观结论可能是错觉，
   默认必须补一轮量化回查，
   而不是只靠更多描述自洽

## 当前项目中的直接落地
1. `step72` 的局部偶发毛刺
   不能只靠听感描述，
   后续需要继续找:
   - 更纯的毛刺指标
   - 更能排除 target 自身瞬态的样本
2. `step60` 的静音段底音泄漏
   当前已具备:
   - 听感支持
   - `mean_active_fraction`
     量化支持
   因而可暂时提升为:
   - 量化支持的结论
3. “更尊重原音量变化”
   目前仍应继续补:
   - activity 对齐类量化指标
   避免只停留在主观印象

## 一句话结论
- 从现在开始，
  主观听审负责提出假设，
  量化回查负责决定结论级别。
- 任何没有量化回查支撑的主观判断，
  默认都不能写成强结论。
