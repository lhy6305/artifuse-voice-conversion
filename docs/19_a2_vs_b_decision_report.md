# `A2` vs `B` 决策报告

## 目的
- 基于 `A1` 首轮实跑结果和当前事件标签体检结果，整理下一步该继续做 `A2` 还是转向 `B`。
- 本报告只做事实收集、优缺点分析和建议，不替用户直接拍板。

先说人话：
- `A1` 已经告诉我们，单靠“再拧一点 event loss”不是很稳。
- 新补的事件标签分析又说明，当前这套 `event_target` 本身就很像“把声学特征换个名字再喂回去”。
- 所以下一步更像是在二选一：
  - 继续小修小补试 `A2`
  - 或承认“老师出的题就不够好”，转去做 `B`

## 本轮新增事实

- `A1` final validation `loss_total = 5.714126`

## `A2` 还能做什么
这里的 `A2` 指继续留在方案 `A` 内，但改成更谨慎的约束试验。

### 可行做法
1. 不再同时做“权重爬升 + 维度重权”
- 拆成单变量实验

2. 缩短权重爬升窗口
- 例如只在更短的 late window 里轻推，不从 `step100` 一路推到 `250`

3. 增加早停门槛
- 一旦 validation 比 baseline 明显恶化，就提前判停

4. 只做维度重权，不改全局 `event` 权重
- 避免再次明显压弱 `z_art`

### 优点
- 成本低
- 很快
- 还能进一步验证“纯训练约束到底还有没有可挖空间”

### 缺点
- 它解决不了当前 `event_target` 本身冗余和错位的问题
- 即使跑出一点提升，也可能仍然只是局部修补

先说人话：
- `A2` 像是在继续调教现在这位老师。
- 便宜，也快。
- 但如果老师本来出的题就不太对，那你再怎么盯纪律，天花板也不会太高。

## `B` 现在为什么更有说服力
这里的 `B` 指优先做事件标签/监督升级。

### 证据链
1. `A1` 证明纯加权不能稳妥解决问题
2. 事件标签分析证明当前监督高度冗余
3. 源/目标事件标签分布存在明显错位
4. 设计稿原本就更偏向：
  - `phone / manner / event boundary`
  - 而不是单纯启发式能量门限

### `B` 的现实含义
- 不一定一上来就做最重的全量重构。
- 可以先做 `B1` 级别的监督升级调研，例如：
  - 明确 8 维事件里哪些维度应保留
  - 哪些维度应拆分或替换
  - 哪些维度应由文本/对齐辅助，而不是只靠声学阈值

### 优点
- 更接近根因
- 更符合设计稿
- 一旦做对，后面所有 `e_evt` 约束实验都会更可信

### 缺点
- 数据工程和标签设计更重
- 不会像 `A2` 那样当天就能再跑一轮训练

先说人话：
- `B` 不是继续拧旋钮，而是先把老师换得更靠谱一点。
- 这条路慢，但从现在证据看，更像治本。

## 我的建议
- 更推荐下一步转向 `B`，而不是继续深挖 `A2`。

### 理由
1. `A1` 已经给出明显负信号
- 继续留在 `A` 内部的收益预期下降了

2. 当前 `event_target` 的冗余证据太强
- 这不是简单调权重就能抹掉的

3. 源/目标分布错位很明显
- 继续放大现有监督，可能只会把错位放大

### 但我不建议完全跳过 `A2` 的一种情况
- 如果你更想用最低成本再确认一次“纯约束还有没有剩余空间”，可以先做一个极小改动版 `A2`：
  - 只做维度重权
  - 不做全局 `event` ramp
  - 加严格 early-stop

这会是一个更便宜的“最后确认”。
